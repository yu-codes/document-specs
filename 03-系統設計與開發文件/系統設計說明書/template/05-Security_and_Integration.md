## 7.2 授權機制

### RBAC 模型

```
使用者 (User) → 角色 (Role) → 權限 (Permission)
```

### 權限定義格式

```
resource:action
例: users:read, users:write, orders:delete
```

### 角色-權限對應

| 權限 | ADMIN | MANAGER | USER | VIEWER |
|------|-------|---------|------|--------|
| users:read | ✓ | ✓(部門) | ✓(自己) | — |
| users:write | ✓ | — | ✓(自己) | — |
| users:delete | ✓ | — | — | — |
| [resource]:read | ✓ | ✓ | ✓ | ✓ |
| [resource]:write | ✓ | ✓ | ✓ | — |
| [resource]:delete | ✓ | ✓ | — | — |

## 7.3 資料加密

| 項目 | 方式 | 說明 |
|------|------|------|
| 傳輸加密 | TLS 1.2+ | 所有 HTTP 通訊 |
| 密碼儲存 | bcrypt | 單向雜湊 |
| 敏感欄位 | AES-256-GCM | 身分證、信用卡號 |
| API Key | HMAC-SHA256 | 第三方整合金鑰 |
| 備份加密 | AES-256 | 資料庫備份 |

## 8.2 整合模式

| 模式 | 使用情境 | 優點 | 缺點 |
|------|---------|------|------|
| 同步 API | 即時查詢 | 即時回應 | 依賴外部服務 |
| 非同步佇列 | 批次處理 | 解耦、重試 | 非即時 |
| Webhook | 事件通知 | 即時推送 | 需驗證來源 |

## 8.3 容錯策略

| 策略 | 說明 | 適用場景 |
|------|------|---------|
| 重試 (Retry) | 指數退避重試 | 暫時性故障 |
| 斷路器 (Circuit Breaker) | 故障達閾值時停止呼叫 | 服務長時間不可用 |
| 降級 (Fallback) | 回傳快取或預設值 | 非關鍵服務 |
| 超時 (Timeout) | 限制等待時間 | 所有外部呼叫 |
| 隔艙 (Bulkhead) | 資源隔離 | 防止連鎖故障 |
